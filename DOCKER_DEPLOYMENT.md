# Docker Deployment Guide - Alwaleed Studio

This guide will help you deploy your Alwaleed Studio application using Docker and Traefik.

## 📋 Prerequisites

- Docker Engine 20.10+
- Docker Compose 2.0+
- Domain name pointed to your server
- Ports 80 and 443 open on your firewall

## 🚀 Quick Start

### 1. Configure Environment Variables

Create a production `.env` file with your actual values:

```bash
# Copy example and edit
cp .env.example .env
nano .env
```

Required variables:
```env
# Supabase
VITE_SUPABASE_URL=your-supabase-url
VITE_SUPABASE_ANON_KEY=your-supabase-anon-key
SUPABASE_URL=your-supabase-url
SUPABASE_SERVICE_KEY=your-supabase-service-key

# Paylink
PAYLINK_APP_ID=your-app-id
PAYLINK_SECRET_KEY=your-secret-key
PAYLINK_TEST_MODE=false

# Frontend URL (production)
FRONTEND_URL=https://yourdomain.com

# Node Environment
NODE_ENV=production
PORT=5000
```

### 2. Update Domain Names

Edit `docker-compose.yml` and replace all instances of:
- `yourdomain.com` with your actual domain
- `api.yourdomain.com` with your API subdomain
- `traefik.yourdomain.com` with your Traefik dashboard subdomain

```yaml
# Find and replace these:
- "traefik.http.routers.backend.rule=Host(`api.yourdomain.com`)"
- "traefik.http.routers.frontend.rule=Host(`yourdomain.com`)"
```

### 3. Configure Traefik Dashboard Authentication

Generate a password hash for the Traefik dashboard:

```bash
# Install htpasswd (if not installed)
sudo apt-get install apache2-utils

# Generate password (replace 'your-password' with your actual password)
echo $(htpasswd -nb admin your-password) | sed -e s/\\$/\\$\\$/g
```

Copy the output and update this line in `docker-compose.yml`:
```yaml
- "traefik.http.middlewares.dashboard-auth.basicauth.users=admin:$$apr1$$abc123$$xyz"
```

### 4. Update Email for Let's Encrypt

In `docker-compose.yml`, update the email for SSL certificates:

```yaml
- "--certificatesresolvers.letsencrypt.acme.email=your-email@example.com"
```

### 5. Build and Deploy

```bash
# Build the images
docker-compose build

# Start all services
docker-compose up -d

# Check logs
docker-compose logs -f

# Check status
docker-compose ps
```

## 🔧 Management Commands

### View Logs

```bash
# All services
docker-compose logs -f

# Specific service
docker-compose logs -f frontend
docker-compose logs -f backend
docker-compose logs -f traefik
```

### Restart Services

```bash
# Restart all
docker-compose restart

# Restart specific service
docker-compose restart backend
docker-compose restart frontend
```

### Stop Services

```bash
# Stop all services
docker-compose down

# Stop and remove volumes (WARNING: This deletes data!)
docker-compose down -v
```

### Update Application

```bash
# Pull latest code
git pull origin main

# Rebuild and restart
docker-compose build
docker-compose up -d

# Or rebuild specific service
docker-compose build backend
docker-compose up -d backend
```

## 🔒 SSL/TLS Certificates

Certificates are automatically generated by Let's Encrypt through Traefik.

**Important:**
- Ensure your domain DNS is properly configured before starting
- Let's Encrypt has rate limits (50 certificates per domain per week)
- For testing, use Let's Encrypt staging:
  ```yaml
  - "--certificatesresolvers.letsencrypt.acme.caserver=https://acme-staging-v02.api.letsencrypt.org/directory"
  ```

View certificate info:
```bash
docker exec alwaleed-traefik cat /letsencrypt/acme.json
```

## 🐛 Troubleshooting

### Containers Won't Start

```bash
# Check logs
docker-compose logs

# Check if ports are in use
sudo netstat -tulpn | grep :80
sudo netstat -tulpn | grep :443
```

### SSL Certificate Issues

```bash
# Check Traefik logs
docker-compose logs traefik

# Verify DNS is pointing to your server
dig yourdomain.com
dig api.yourdomain.com
```

### Backend Can't Connect to Supabase

```bash
# Check backend logs
docker-compose logs backend

# Verify environment variables
docker exec alwaleed-backend env | grep SUPABASE
```

### Frontend Shows 502 Bad Gateway

```bash
# Check backend is running
docker-compose ps backend

# Check backend health
docker exec alwaleed-backend wget -O- http://localhost:5000/health

# Check Traefik routing
docker-compose logs traefik | grep backend
```

## 🔄 Backup and Restore

### Backup Volumes

```bash
# Backup uploads
docker run --rm -v alwaleed-backend_backend-uploads:/data -v $(pwd):/backup alpine tar czf /backup/uploads-backup.tar.gz /data

# Backup certificates
docker run --rm -v alwaleed-backend_traefik-certificates:/data -v $(pwd):/backup alpine tar czf /backup/certificates-backup.tar.gz /data
```

### Restore Volumes

```bash
# Restore uploads
docker run --rm -v alwaleed-backend_backend-uploads:/data -v $(pwd):/backup alpine tar xzf /backup/uploads-backup.tar.gz -C /

# Restore certificates
docker run --rm -v alwaleed-backend_traefik-certificates:/data -v $(pwd):/backup alpine tar xzf /backup/certificates-backup.tar.gz -C /
```

## 📈 Scaling

### Horizontal Scaling (Multiple Replicas)

```bash
# Scale backend to 3 instances
docker-compose up -d --scale backend=3
```

Traefik will automatically load balance between instances.

### Vertical Scaling (Resource Limits)

Add to `docker-compose.yml`:

```yaml
backend:
  # ... existing config ...
  deploy:
    resources:
      limits:
        cpus: '2'
        memory: 2G
      reservations:
        cpus: '0.5'
        memory: 512M
```

## 🔐 Security Best Practices

1. **Never commit `.env` file** - It's in `.gitignore`
2. **Use strong passwords** for Traefik dashboard
3. **Keep Docker updated**:
   ```bash
   sudo apt update && sudo apt upgrade
   ```
4. **Regularly update images**:
   ```bash
   docker-compose pull
   docker-compose up -d
   ```
5. **Monitor logs** for suspicious activity
6. **Enable firewall**:
   ```bash
   sudo ufw allow 80/tcp
   sudo ufw allow 443/tcp
   sudo ufw enable
   ```
